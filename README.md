# ⚔ LoL Picks Manager

Профессиональный инструмент для управления пиками чемпионов на турнирах по League of Legends.

---

## Возможности

- 📋 **Пресеты** для разных команд (Наша команда / Противники)
- 🎯 **Роль (позиция)**: Топ, Лес, Мид, Адк, Сап
- ⚔ **Класс урона**: АД, АП, Танк, Утилити, Гибрид — каждый с цветовым кодом текста на чёрном чипе
- 🔍 **Подробный тип урона**: Физический, Критический, Burst, DOT, Engage и т.д. — зависит от класса урона
- 🔢 **Сортировка**: по роли (Топ→Лес→Мид→Адк→Сап), затем по классу урона (АД→АП→Танк→Утилити→Гибрид), затем по тиру
- 🔍 **Фильтрация** по роли, классу урона, тиру (S/A/B/C) и поиску по имени
- 🗡️ **Билд**: основные + ситуативные предметы с иконками
- 💾 **Сохранение**: локально в JSON или в Google Sheets
- ☁ **Google Sheets**: общая таблица для всей команды через сервисный аккаунт — без браузерной авторизации, без пересылки файлов между игроками

## Роли и классы урона

| Роль (позиция) | | Класс урона | Цвет текста чипа |
|---|---|---|---|
| Топ  | | АД        | 🔴 Красный |
| Лес  | | АП        | 🔵 Синий |
| Мид  | | Танк      | 🟢 Зелёный |
| Адк  | | Утилити   | 🟠 Оранжевый |
| Сап  | | Гибрид    | 🟣 Фиолетовый |

Фон у всех чипов класса урона — чёрный, отличается только цвет обводки и текста.
Подробный тип урона (физический/критический/burst и т.д.) выводится отдельной колонкой и зависит от выбранного класса урона.

## ☁ Google Sheets: общая таблица для команды

Используется **сервисный аккаунт** Google — специальный "бот"-аккаунт, который не требует входа через браузер. Поэтому подключение происходит мгновенно у всех членов команды, а не виснет в ожидании авторизации.

### Настройка (делает один человек — капитан/аналитик)

1. Перейди на [console.cloud.google.com](https://console.cloud.google.com)
2. Создай проект, включи **Google Sheets API** и **Google Drive API**
3. **IAM и администрирование → Сервисные аккаунты → Создать сервисный аккаунт**
4. Открой созданный аккаунт → вкладка **Ключи → Добавить ключ → JSON**
5. Скачанный файл переименуй в `service_account.json` и положи рядом с `main.py`
6. Открой Google-таблицу → **Поделиться** → вставь email сервисного аккаунта (вида `xxx@yyy.iam.gserviceaccount.com`, виден внутри JSON-файла в поле `client_email`, а также в диалоге программы после выбора файла) → роль **Редактор**
7. Скопируй ID таблицы из адресной строки браузера:
   ```
   https://docs.google.com/spreadsheets/d/  ВОТ_ЭТО_ID  /edit
   ```
8. В программе: **☁ Google Sheets → Управление** → вставь ID в поле «ID таблицы» → **Сохранить ID таблицы** → **Подключиться**

### Раздача доступа команде

Каждому игроку нужно передать (вне публичных чатов и репозиториев):
- файл `service_account.json`
- (ID таблицы уже зашит в `config.json` внутри программы, либо тоже передаётся отдельно)

После этого у всех — **одна и та же таблица**, видна сразу при запуске программы (есть автоподключение при старте, если файлы на месте).

### ⚠️ Безопасность ключа

`service_account.json` — это секретный ключ с правами на запись в таблицу. Никогда не публикуй его в публичных репозиториях, чатах или скриншотах. Если ключ "засветился" — зайди в Google Cloud Console → Сервисные аккаунты → твой аккаунт → **Ключи** → удали скомпрометированный ключ и создай новый.
